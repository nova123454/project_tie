미래는 우리들의 손 안에

우리들은 울기 위해 태어난 게 아니야

제가 가장 종아하는 작품 역경무뢰 카이지에 나오는 주인공의 대사입니다.

내 인생은 아직 오지 않았어! 사람은.. 가짜로 살고 있을 수도 가짜로 죽을 수도 없어.. 당연하지 문제는 그 당연한 사실을 깨닫고 있느냐야.

이세돌 신의 78수에 알파고가 무너지는 순간입니다. 저는 저에게 신의 한 수를 두고 싶습니다.

Frontend

개발자를 목표로 공부하고 있는 윤동일입니다.

모든 영웅조차 다 자신의 한계가 있었다. 동서양 영원한 고전 삼국지에 나오는 말입니다.

하물며 평범한 저에게 인생의 한계는 항상 높았고, 좌절을 반복해왔던 거 같습니다.
하지만 부정적인 태도로 어제와 같은 내일을 관성적으로 무의미하게 살아가는 대신,
한계를 극복하고자 했던 영웅들의 삶을 등대삼아서 도전해보고 싶습니다.

React를 처음 배워서 사용해봤습니다.

1. React를 활용하여 스타벅스 홈페이지 느낌을 표현해봤습니다.

2. JSX 문법을 통해 리액트 레이아웃을 만들었습니다.

3. 리액트 라우터로 상세 페이지 (Detail,Login,Product)를 구현해봤습니다.

4. 리액트 라이브러리 Redux를 사용하여 store 저장소에서 component state를 관리해봤습니다.

5. 리액트 부트스트랩을 활용하여 Submenu창을 만들어봤습니다.

6. git에 package.json 라인 경로로 배포해봤습니다.






앞으로의 게획

앞으로 React 및 자바스크립트, nodejs를 중점적으로 더 공부하여
좋은 웹사이트를 만들 수 있는 실력을 키우도록 하겠습니다.



AJAX 오픈 API로 도서 홈페이지를 만들었습니다.

1. Open API를 사용하여 도서정보를 가져왔습니다.

2. jQuery 문법을 사용하여 슬라이드를 구현했습니다.

3. JSON으로 데이터를 만들었고, 정보를 가져왔습니다.

4. HTML CSS 문법을 준수하여, 실제 홈페이지 디자인과 비슷하게 만들어봤습니다.

5. SCSS/SASS 기본 문법을 사용하여 만들었습니다.






앞으로 발전시키겠습니다.

HTML 문법에 따라서 레이아웃을 짜고,
CSS FLEX/GRID를 사용해서,
실제 홈페이지와 같은 디자인을 하도록 연습하겠습니다.





팀 프로젝트로 전시회 홈페이지를 만들었습니다.

1. 자바스크립트를 배워서 자바스크립트로 홈페이지에 역동적 기능을 줬습니다.

2. 팀원들과 소통하며 폰트 및 사진을 어떤식으로 배치할 지를 논의해봤습니다.

3. 미디어 쿼리를 사용하여 사용자가 편하게 사용할 반응형 웹페이지를 구현했습니다.

4. css 애니메이션으로 버튼에 다양한 효과를 주었습니다.







앞으로의 게획

같이 공부했던 3분과 코드를 짜서 홈페이지를 만들어보면서,
어떻게든 원리를 이해해서, 자신만의 코드를 짜야한다는 걸 느꼈습니다.
웹개발자가 되려면 시행착오를 통해서 자신만의 코드를 가져야겠다고 생각했습니다.
앞으로 계속 수정하고, 공부해서 도전해보겠습니다.

HTML/CSS를 배워서 만들어봤습니다.

1. 반응형 웹으로 만들었습니다. 미디어 쿼리를 3부분으로 나눴습니다.

2. HTML를 웹 사용에 불편하신 분들도 잘 사용할 수 있게 목차를 만들어보려고 노력했습니다.

3. 포토샵을 이용해서 이미지를 수정하여 배치해봤습니다.

4. 헤더, 푸터를 알맞게 배치하여 레이아웃을 잡아봤습니다.

5. 잘 만들어진 웹사이트를 눈으로 보면서, 어떤 식으로 웹사이트를 만들어할 지 생각해봤습니다.






앞으로의 게획
